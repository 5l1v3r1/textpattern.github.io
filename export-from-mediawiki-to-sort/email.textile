h4(#example_4with_symbolset_s_email_glyph). Example 4: With Symbolset's &quot;email&quot; glyph

If you happen to use the “email” glyph in the social media set of "Symbolset":http://symbolset.com/, you can still use this tag. Let's say you're creating a social button bar using Symbolset glyphs in a list. The normal way to do this would be to set up your selectors on the individual anchor elements, like the first three list items show below. But for the email glyph you need to put the selectors in the

bc(code). li


since you can't put them in the

bc(code). a


, as the last list item shows:

bc(code). <ul class="socbar">
    <li><a href="https://twitter.com/xxx" class="ss-icon twit" title="">twitter</a></li>
    <li><a href="https://plus.google.com/xxx/posts" class="ss-icon gplus" title="">googleplus</a></li>
    <li><a href="http://www.linkedin.com/in/xxx" class="ss-icon in" title="Pffft">linkedin</a></li>
    <li class="ss-icon email"><txp:email email="your@email.tld" linktext="email" title="" /></li>
</ul>


If you're using Symbolset, then you'll know that the

bc(code). linktext=""


attribute in the last list item above _has_ to be “*email*” for the glyph to work.

Then the CSS must be like this to target both instances of selector use...

bc(code). /* 
****** First the common rules
*/
a.ss-icon,
li.ss-icon a { /* design as you want */ }
/*
****** Target each one if specific hover effect is wanted.
*/
.twit:hover { /* design as you want */ }
... etc ...
li.email a:hover { /* design as you want */ }


*Tip:* See the "feed&#95;link":/home/www/zendstudio/dokuwiki/bin/doku.php?id=feed_link tag for a similar solution for Symbolset's “rss” glyph.
