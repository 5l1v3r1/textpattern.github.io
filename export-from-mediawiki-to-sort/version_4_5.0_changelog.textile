\nChanges in 4.5.0

* Minimum system requirement: PHP 5.2
* Modern default site theme (responsive, HTML5, CSS3)
* Choice of HTML5 or XHTML document type as a preference
* Textile 2.4.1. See full changelog at "$":http://textpattern.com/textile-changelog
* User interface realignment and clean-up
* Additional admin theme (Hive)
* Beautified setup process
* Write, Plugin and Section panels save some changes without page refreshes (AJAX)
* WordPress import tool imports image media attachemnts as article images, can use 'utf8' or 'latin1' source databases
* New advanced preference “Login info email address” allows to define a 'From:' address from which a new user's login info is sent
* Check for new Textpattern version moved from Preferences to Diagnostics panel
* Removed raw PHP support in pages and articles
* Plugin status is preserved across plugin updates
* &lt;txp:link&#95;to&#95;prev&gt; and &lt;txp:link&#95;to&#95;next&gt; adhere to the sort order set by &lt;txp:article&gt;
* Deprecated 'escape' attribute for &lt;txp:search&#95;term /&gt;
* Removed 'align' attribute for &lt;txp:image /&gt;, &lt;txp:thumbnail /&gt;, &lt;txp:article&#95;image /&gt;
* &lt;txp:css /&gt; accepts a comma-separated list of style-sheet names for the 'name' attribute
* &lt;txp:txp&#95;die /&gt;: New attribute 'url'. Specifies location target for HTTP stati 301, 302, 307
* &lt;txp:article&#95;custom&gt;, &lt;txp:file&#95;download&#95;list&gt; preserve sorting order from 'id' attribute
* &lt;txp:images /&gt; preserves sorting order from article image ids
* &lt;txp:file&#95;download&gt; may be used as a container tag
* &lt;txp:comment&#95;form /&gt;: New attributes 'previewlabel', 'submitlabel', 'rememberlabel', 'forgetlabel'
* New &lt;txp:author&#95;email&gt; tag
* &lt;txp:linklist /&gt; accepts a comma-separated list of link ids in the new 'id' attribute
* &lt;txp:images /&gt; uses the sort order of images from the 'id' attribute if present.
* Developer: Introduce gTxtScript() to help with the l10n of client-side strings
* Developer: Introduce txpspecialchars() as a recommended HTML5-safe alternative to htmlspecialchars()
* Developer: New events 'article&#95;saved', 'article&#95;posted', 'comment.saved', 'article&#95;ui'.'partials&#95;meta', 'article&#95;ui'.'validate&#95;save', 'image&#95;ui'.'fullsize&#95;image', 'image&#95;ui'.'thumbnail&#95;image', 'admin&#95;criteria'.'author&#95;list', 'admin&#95;criteria'.'author&#95;list', 'admin&#95;criteria'.'css&#95;list', 'admin&#95;criteria'.'discuss&#95;list', 'admin&#95;criteria'.'file&#95;list', 'admin&#95;criteria'.'form&#95;list', 'admin&#95;criteria'.'image&#95;list', 'admin&#95;criteria'.'link&#95;list', 'admin&#95;criteria'.'list&#95;list', 'admin&#95;criteria'.'log&#95;list', 'admin&#95;criteria'.'page&#95;list', 'admin&#95;criteria'.'section&#95;list' (NB: 'image&#95;ui'.'image&#95;edit' and 'image&#95;ui'.'thumbnail&#95;edit' changed)
* Developer: Introduce textpattern.Relay, a pub/sub hub for client-side events
* Developer: Introduce txpAsyncForm (a jQuery plugin for asynchronous posts from forms) and txpAsyncHref (a jQuery plugin for asynchronous posts from links)
* Developer: Introduce safe&#95;escape(), Constraint() and Validator() classes plus their descendents, callback&#95;event&#95;ref(), theme::announce&#95;async()
* Developer: Restrict plugin type '3' to load only at non-AJAX requests. Introduce plugin type '4' to be loaded on the admin side for both AJAX and non-AJAX requests. Introduce plugin type '5' to be loaded on the public side and on the admin side for both AJAX and non-AJAX requests.
* Developer: Run custom post-update code from txpath.'/update/custom/post-update&#42;.php' if this glob() exists
* Developer: Include custom code like page-caches before the page is assembled by setting $txpcfg['pre&#95;publish&#95;script'] to a valid filename.

* Developer: Additional plugin types '4' (admin-side, only for AJAX requests), and '5' (public side; admin-side, only for AJAX requests).
* Security: Admin-side disallows framing, sends “X-Frame-Options: SAMEORIGIN” header.
* Security: The 'txp&#95;login' cookie is set with a 'HttpOnly' attribute.
