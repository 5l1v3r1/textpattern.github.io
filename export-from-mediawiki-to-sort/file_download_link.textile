The "file&#95;download&#95;link":/home/www/zendstudio/dokuwiki/bin/doku.php?id=file_download_link tag is both a _single_ tag and a _container_ tag. Thus it may be used as an opening and closing pair:

bc(code). <txp:file_download_link>
...containing statements...
</txp:file_download_link>


When used as a single tag, Textpattern will replace the tag with a download link to the file being downloaded. As a container, it will assign the link to the given text or tag, while the single tag outputs the file's plain URL.

h3(sectionedit1#attributes). Attributes

Tag will accept the following attributes (*case-sensitive*):

*"filename":/home/www/zendstudio/dokuwiki/bin/doku.php?id=attributes_cross-reference#filename=“text”*
 Name of the file to download. *"id":/home/www/zendstudio/dokuwiki/bin/doku.php?id=attributes_cross-reference#id=“integer”*
 Numeric id of the file to download.

Note:“id” takes precedence over “filename”. If neither is defined and the tag is not used within the context of a file, nothing is returned.
