\nBelow is a template that all plugin developers should adhere to as closely as possible. Using a template makes writing the plugin help easier, more consistent and more complete, as well as easier for your users to understand and navigate.

*Do not use inline CSS styling* of your own - please make use of the existing CSS stylesheet that was provided by the admin theme instead.

The following guidelines are taken from "this forum post":http://forum.textpattern.com/viewtopic.php?id=25327

h3(sectionedit1#textile_tags). Textile tags

Use the following Textile tags when formatting help files:

<pre class="code">
h1. Plugin title

h2. Section name

h3. Subsection name

p. Body text uses regular paragraph formatting

p. An %(highlight)important statement% within a paragraph

# A number list example
# Item 2
# Item 3

* A bullet list example
* Item 2
* Item 3

An inline @code@ tag

bc. A pre code box with pre formatted
  code examples
</pre>

h3(sectionedit2#suggested_sections). Suggested sections

Here are the suggested sections for your help file - use as many as is necessary for your help file:

# Summary and version details
# Table of contents
# List of features
# Plugin requirements
# Installation (when differs from standard installation procedure)
# Uninstallation (when differs from standard uninstallation procedure)
# Tags (along with attribute descriptions and examples)
# Exposed functions (for class plugins)
# Global variables
# Plugin events
# Examples
# License
# Author contact
